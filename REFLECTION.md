# Reflection

## Project Overview

This project was designed to simulate a real SaaS startup workflow focused on AI cost optimization for teams.

The goal was not only to build a working product, but also to demonstrate:
- engineering thinking
- product design
- deployment workflows
- scalability considerations

---

# What Went Well

## Full-Stack Development
Successfully integrated:
- frontend
- backend APIs
- database
- deployment infrastructure

This project significantly improved my understanding of modern SaaS architecture.

---

## Rapid Iteration
The project evolved quickly from:
- simple audit form
to
- fully deployed SaaS MVP

This iterative workflow mirrored real startup product development.

---

## Deployment Workflow
Using:
- GitHub Actions
- Vercel
- Supabase

helped simulate a real production workflow.

---

# Biggest Challenges

## API Integration
Integrating OpenAI APIs while handling quota failures required robust fallback handling.

---

## Database Security
Understanding Supabase RLS policies and debugging insert permissions was a major learning experience.

---

## Dynamic Routing
Building public shareable reports using dynamic routes required deeper understanding of Next.js App Router patterns.

---

# Key Learnings

- Full-stack systems require careful integration between frontend, backend, and database layers.
- Graceful failure handling is critical for production-ready applications.
- CI/CD pipelines significantly improve development reliability.
- Documentation is as important as implementation in real engineering environments.

---
